# 정처기개념 주관식 전체문제

---

## I. 소프트웨어 구축

### 소프트웨어 생명 주기(SDLC)

1. 소프트웨어 생명 주기의 일반적인 6단계 순서는?
정답: 프로젝트 계획, 요구 분석, 설계, 구현, 테스트, 유지 보수

2. 선형 순차적 개발로 고전적 전통적 개발 모형이라 불리는 Step-by-Step 방식은?
정답: 폭포수(Waterfall) 모형

3. 고객의 need 파악을 위해 견본/시제품을 통해 최종 결과 예측하고 인터페이스 중심으로 요구사항 변경이 용이한 모형은?
정답: 프로토타입(Prototype) 모형

4. 폭포수 + 프로토타입 + 위험 분석 기능이 추가되어 위험 관리/최소화하는 모형은?
정답: 나선형(Spiral) 모형

5. 나선형 모형의 반복 순서 4단계는?
정답: 계획 수립, 위험 분석, 개발 및 검증, 고객 평가

6. 일정한 짧은 주기(Sprint 또는 Iteration)를 반복하며 개발 진행하고 고객 요구사항에 유연하게 대응하는 모형은?
정답: 애자일(Agile)

7. 애자일 방법론의 예시 5가지는?
정답: XP, Scrum, FDD(기능중심), 린(LEAN), DSDM

8. 최상위 컴포넌트 설계 후 하위 기능을 부여하는 절차 지향 설계 방식은?
정답: 하향식(Top-down) 설계

9. 최하위 모듈 먼저 설계 후 이들을 결합하고 검사하는 객체 지향 설계 방식은?
정답: 상향식(Bottom-up) 설계

10. 명백한 역할을 가지며 재사용되는 모듈 단위로 인터페이스 통해 접근 가능한 것은?
정답: 컴포넌트(Component)

### 익스트림 프로그래밍(XP)

11. 고객의 요구사항을 유연하게 대응하기 위해 고객 참여와 신속한 개발 과정을 반복하는 방법론은?
정답: 익스트림 프로그래밍(XP)

12. XP의 5가지 핵심 가치는?
정답: 용기, 단순성, 의사소통, 피드백, 존중

13. 개발 과정에서 즉각적인 피드백을 받아 문제를 조기에 수정하는 XP 핵심 가치는?
정답: 피드백

14. XP의 기본 원리 중 두 명이 함께 코딩하는 것은?
정답: 짝(Pair) 프로그래밍

15. XP의 기본 원리 중 코드 구조 개선하면서 기능은 변경하지 않는 것은?
정답: 디자인 개선(리팩토링)

16. XP에서 모듈을 팀 전체가 공동으로 소유하는 원리는?
정답: 공동 소유권(Collective Ownership)

17. XP에서 테스트를 먼저 작성한 뒤 코드를 구현하는 기본 원리는?
정답: 테스트 주도 개발(TDD)

18. XP에서 개발 문서보다 소스코드에 중점을 두어 문서화 하지 않는 방식은?
정답: 문서화 생략

### 프로젝트 계획 - 비용 산정

19. 조직 내 두 명 이상의 전문가에게 비용 산정을 의뢰하는 하향식 기법은?
정답: 전문가 감정 기법

20. 한 명의 조정자와 여러 전문가의 의견을 종합하여 전문가 감정 기법의 주관적 편견 보완하는 기법은?
정답: 델파이 기법

21. 프로젝트 세부 작업 단위별로 비용 정산 후 전체 비용을 산정하는 방식은?
정답: 상향식 비용 산정 기법

22. 코드 라인 총 수를 기반으로 비용 산정 공식 (a+4c+b)/6 을 사용하는 기법은?
정답: LOC(Source Line of Code)

23. LOC 기법에서 a, b, c는 각각 무엇인가?
정답: a=낙관치, b=비관치, c=기대치

24. 생명 주기 각 단계별로 인월 수를 산정하여 LOC 기법을 보완한 기법은?
정답: 개발 단계별 인월 수(Effort Per Task)

25. 보헴(Boehm)이 제안하고 원시코드 라인 수 기반의 수학적 비용 산정 기법은?
정답: COCOMO(Constructive Cost Model)

26. COCOMO에서 중소규모 SW용 50KDSI 이하인 유형은?
정답: 조직형(Organic)

27. COCOMO에서 300KDSI 이하의 트랜잭션 처리 시스템 유형은?
정답: 반분리형(Semi-detached)

28. COCOMO에서 300KDSI 이상의 최대형 규모 SW 관리 유형은?
정답: 내장형(Embedded)

29. Rayleigh Norden 곡선 노력 분포도 기초의 비용 산정 모형은?
정답: PUTNAM

30. PUTNAM 모형 기초로 개발된 자동화 추정 도구는?
정답: SLIM

31. SW 기능 증대 요인에 가중치 부여 후 합산하여 기능점수 산출하는 기법은?
정답: Function Point(FP)

32. FP 모형 기반으로 개발된 자동화 추정 도구는?
정답: ESTIMACS

### 개발 일정 산정

33. 프로젝트 목표 달성을 위한 활동과 업무를 세분화하는 기법은?
정답: WBS(Work Breakdown Structure)

34. 프로젝트 작업 상호관계를 네트워크로 표현하고 불확실한 상황에 사용하는 기법은?
정답: PERT

35. PERT에서 원 노드(작업)와 간선(화살표)으로 구성되고 낙관치/기대치/비관치를 기재하는 것은?
정답: PERT

36. 미국 Dupont 회사 개발, 확실한 상황에 사용하며 노드(작업)인 네트워크 기법은?
정답: CPM(Critical Path Method)

37. 네트워크에서 경로상 가장 오래 걸리는 시간으로 전체 프로젝트 최소 소요 기간이 되는 경로는?
정답: 임계 경로(Critical Path)

38. 각 작업의 시작/종료 일정을 막대 바 도표로 표현하는 시간선 차트는?
정답: 간트 차트(Gantt Chart)

39. 간트 차트의 단점 두 가지는?
정답: 작업 경로 표현 불가, 계획 변화에 대한 적응성 낮음

정답: 비기능적 요구사항

정답: 요구사항 도출/추출

### 객체지향 분석 모델

64. 럼바우의 객체 모델링에서 객체 다이어그램으로 객체들 간 관계를 규정하는 것은?

66. 럼바우의 기능 모델링에서 DFD로 다수 프로세스 간 처리 과정을 표현하는 것은?

### UML

85. 고객/개발자 간 원활한 의사소통을 위해 표준화한 객체지향 모델링 언어는?
정답: UML(Unified Modeling Language)

86. UML의 구성요소 세 가지는?
정답: 사물(Things), 관계(Relationship), 다이어그램(Diagram)

87. UML에서 2개 이상 사물이 서로 관련된 관계는?
정답: 연관 관계(Association)

88. UML에서 전체-부분 관계를 나타내는 관계는?
정답: 집합 관계(Aggregation)

89. UML에서 집합 관계 내 한 사물 변화가 다른 사물에 영향을 주는 관계는?
정답: 포함(합성) 관계(Composition)

90. UML에서 상위 개념일 때 표현하는 일반/구체적 관계는?
정답: 일반화 관계(Generalization)

91. UML에서 한 클래스가 다른 클래스 기능을 사용하는 영향 관계는?
정답: 의존 관계(Dependency)

92. UML에서 한 객체가 다른 객체에게 오퍼레이션 수행을 지정하는 관계는?
정답: 실체화 관계(Realization)

93. UML 구조/정적 다이어그램 암기어 클객컴배복패는?
정답: 클래스, 객체, 컴포넌트, 배치, 복합체 구조, 패키지

94. UML 행위/동적 다이어그램 암기어 유시커상활타상은?

583. 백도어 탐지 방법(무결성 검사, 로그 분석, SetID 파일, 열린 포트 검사)은?

585. 특정 개인/조직 타겟 침입 후 내부 서버 제어권 획득하는 지능형 지속 공격은?

592. 서버 접속 클라이언트 세션 정보 가로채어 강제 종료 후 침입하는 공격은?

정답: 웨일링(Whaling)

598. 키 입력 캡처 프로그램으로 개인정보 빼내는 행위는?

599. 할당 메모리 범위 넘어선 위치에서 자료 사용 시 발생하는 것은?

601. 가능한 모든 문자열 조합으로 암호키 탈취하는 공격은?

603. 블루투스 연결 취약점 이용 공격은?
정답: Bluebug

604. 수집한 로그인 정보를 다른 사이트 무작위 대입하는 공격은?

605. 타겟 기업 침해 위해 협력사/파트너 네트워크 해킹하는 것은?
정답: Island Hopping

606. 인간 신뢰 바탕으로 사람들 속여 보안 절절 깨뜨리는 비기술적 침입 수단은?
정답: 사회 공학(Social Engineering)

607. 권한 없는 영역 접근하여 시스템 제어하는 악성 SW 모음은?
정답: Rootkit

608. 정보보호 관리 체계로 정보 자산 안전 보호 위한 절차/대책 확보하는 것은?
정답: ISMS(Information Security Management System)

609. 특정 목적으로 수집되어 활용되지 않고 저장만 된 대량 데이터는?

610. 가짜 바이러스 경고/시스템 문제 표시하여 돈/소프트웨어 설치 속이는 것은?
