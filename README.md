# STUDY_VIEWER

STUDY_VIEWER는 정보처리기사 실기처럼 짧은 문항과 정답을 반복해서 보는 로컬 학습 뷰어입니다. Markdown 또는 텍스트 파일을 불러오면 문항 카드로 변환하고, 단건보기, 모두보기, 시험보기, 통계를 한 화면에서 다룰 수 있습니다.

## 주요 기능

- `답:` 또는 `정답:` 구문이 있는 Markdown/TXT 파일을 문항 카드로 변환
- Markdown 표 형식 문항도 행 단위로 자동 카드화
- 폴더 클릭 시 폴더 안 모든 파일의 문항을 통합 로딩
- 순차/무작위 정렬 전환, 무작위 전환 때마다 새 셔플 적용
- 단건보기 정답 마스킹, 클릭 재마스킹, 더블클릭 원문 보기
- 정답 수정 및 시험보기 채점에 수정 정답 반영
- 시험보기 답안 임시 저장, 제출 결과 저장, 통계 확인
- OpenCode 스타일 낮 모드와 Resend 스타일 밤 모드 지원
- 문항 카드와 코드 문항을 코드 윈도우 스타일로 표시

## 문항 작성 예시

```md
1. RFC 1321, 128비트 해시 알고리즘은?
정답: MD5

2. 프로토콜의 기본 요소 3가지는?
답: 구문, 의미, 순서
```

표 형식도 사용할 수 있습니다.

```md
| 번호 | 유형 | 핵심 내용 | 정답 |
|---|---|---|---|
| 1 | 보안 | 출발지/목적지 동일 조작 DoS | Land Attack |
| 2 | 해시 | RFC 1321, 128비트 해시 | MD5 |
```

## 로컬 실행

```bash
bun install
bun run dev
```

프로덕션 빌드는 다음 명령으로 생성합니다.

```bash
bun run build
bun run preview
```

## 배포

이 저장소는 GitHub Pages 경로인 `/study-viewer/` 기준으로 빌드되도록 설정되어 있습니다. `dist/` 폴더가 함께 커밋되므로 Pages에서 `dist`를 배포 대상으로 잡으면 정적 사이트로 실행할 수 있습니다.

저장 데이터는 서버가 아니라 브라우저 `localStorage`에 저장됩니다. 같은 브라우저에서는 파일, 답안, 시험 결과가 유지되지만 다른 기기와 자동 동기화되지는 않습니다.

## 포함 자료

`doc/` 폴더에는 학습용 Markdown/HTML 자료가 포함되어 있습니다.

- `정보처리기사_실기_개념요약본.md`

- `정보처리기사_실기_학습자료.html`
